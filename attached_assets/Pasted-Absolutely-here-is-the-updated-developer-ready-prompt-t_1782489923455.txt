# **DEVELOPER PROMPT (Admin‑Controlled Countdown With Multi‑Day Support)**

**Objective:**  
Build an admin‑controlled countdown and reactivation system. The admin must be able to manually trigger the countdown, set the reactivation deposit amount, and edit all displayed messages. The countdown must support durations from minutes up to 4–5 days, and must display in **hours : minutes : seconds** format at all times.

---

## **1. Admin Panel Requirements**

Create an admin interface with the following fields:

### **A. Countdown Activation**
- Toggle: **Activate Countdown** (ON/OFF)
- Dropdown or user selector: **Select User / Account**
- When ON:
  - Countdown begins immediately for the selected user
  - User sees the portal countdown message

### **B. Countdown Duration (Flexible Time Input)**
Admin must be able to set:
- **Days** (0–5 max)
- **Hours**
- **Minutes**

System must convert this into a live countdown displayed as:
**DD days HH:MM:SS**  
or  
**HH:MM:SS** when under 24 hours.

### **C. Reactivation Deposit Amount**
- Numeric input  
- Required  
- Admin‑editable at any time

### **D. Editable Message Fields**
Admin must be able to edit:
- Portal Countdown Message  
- Email Notification Message  
- Reactivation Page Message  
- Closure Explanation Message  

All fields must support multiline text.

---

## **2. Countdown Behaviour**

When admin activates the countdown:

- Display a **live countdown timer** on the user’s portal  
- Timer must show **days, hours, minutes, seconds**  
- User actions **cannot cancel or pause** the countdown  
- When countdown reaches zero:
  - Force‑close the session  
  - Reset the withdrawal pathway  
  - Void NDA seals, declarations, and validations  
  - Redirect user to the Reactivation Deposit Page  

---

## **3. Default Portal Countdown Message (Editable by Admin)**

“Due to extended inactivity, your portal session and withdrawal pathway are scheduled to close when this countdown ends. Once closed, your withdrawal structure will be reset and all NDA seals, declarations, and prior validations will be voided. A reactivation deposit will be required to restore access. You will be redirected to the reactivation deposit page after closure.”

---

## **4. Default Email Notification (Editable by Admin)**

“Your account has remained inactive for an extended period, and your portal session is now scheduled to close. When the countdown ends, your withdrawal pathway will be reset and all NDA seals, declarations, and compliance validations will be voided. To continue with your withdrawal, you will need to complete the reactivation process on the reactivation deposit page that will appear after closure.”

---

## **5. Reactivation Deposit Page Requirements**

After session closure, redirect user to:

`/reactivation-deposit`

The page must display:

- Reason for reactivation  
- Admin‑configured reactivation deposit amount  
- Steps required to restore withdrawal pathway  
- Confirmation message after successful reactivation  

All text must be admin‑editable.

---

## **6. System Logic Requirements**

- Countdown duration supports:
  - Minutes  
  - Hours  
  - Days (up to 4–5 days max)  
- Countdown must always display in **HH:MM:SS** (or DD:HH:MM:SS when applicable)  
- All messages must be editable in admin panel  
- Reactivation deposit amount must be admin‑configurable  
- Reset logic must clear:
  - Withdrawal structure  
  - NDA seal status  
  - Declaration status  
  - Pending validations  
- Redirect must be automatic and non‑bypassable  
- System must log:
  - Countdown activation time  
  - Admin who activated it  
  - User affected
