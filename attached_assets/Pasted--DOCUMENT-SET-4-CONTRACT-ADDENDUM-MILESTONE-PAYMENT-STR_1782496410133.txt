# **SECTION 2 — DEFINITIONS**

### **2.1 “Project”**  
Refers to the IBCCF system as defined in the Master Specification.

### **2.2 “Deliverables”**  
All code, documentation, configurations, assets, and materials produced by the Developer.

### **2.3 “Milestones”**  
The five delivery checkpoints defined in Section 7.

### **2.4 “Acceptance”**  
Formal written confirmation by the Client that a Milestone meets all requirements.

### **2.5 “Defect”**  
Any deviation from the Master Specification, SRS, or expected system behavior.

### **2.6 “Gap”**  
Any missing feature, missing email, missing validation, missing security control, or missing workflow identified in the Master Specification.

# **SECTION 3 — INTELLECTUAL PROPERTY OWNERSHIP**

### **3.1 Full Assignment of Rights**
All Deliverables produced under this contract are the **exclusive property of the Client**.

Developer assigns to the Client:

- All copyrights  
- All code ownership  
- All design rights  
- All documentation rights  
- All derivative rights  
- All deployment rights  
- All commercial rights  

### **3.2 No Retained Rights**
Developer retains **no rights** to:

- Reuse code  
- Resell components  
- Repurpose architecture  
- Publish system details  
- Claim authorship  

### **3.3 Work‑for‑Hire**
All Deliverables are classified as **work‑for‑hire**.

# **SECTION 4 — CONFIDENTIALITY & NON‑DISCLOSURE**

### **4.1 Confidential Information**
Includes:

- System architecture  
- Codebase  
- Database schema  
- Compliance logic  
- Withdrawal logic  
- Countdown logic  
- Reactivation logic  
- Email templates  
- Admin workflows  
- User workflows  
- AI logic  
- Monitoring logic  
- All documents in Sets 1–5  

### **4.2 Developer Obligations**
Developer must not:

- Disclose any confidential information  
- Share screenshots  
- Share code snippets  
- Discuss system logic  
- Store data outside approved systems  
- Upload code to public repositories  
- Use project data for training or testing  

### **4.3 Duration**
Confidentiality obligations last **indefinitely**, even after contract termination.

# **SECTION 5 — NON‑CIRCUMVENTION**

Developer agrees not to:

- Contact users  
- Contact case holders  
- Contact partners  
- Contact investors  
- Contact any third party connected to the project  
- Replicate the system for another client  
- Build a competing system  

Violation constitutes immediate breach.

# **SECTION 6 — DELIVERY OBLIGATIONS**

Developer must:

- Deliver all Milestones on time  
- Close all Gaps  
- Implement all automated emails  
- Implement all countdown logic  
- Implement all reactivation logic  
- Implement all admin workflows  
- Implement all user workflows  
- Implement all security controls  
- Implement all monitoring endpoints  
- Provide written confirmations for each checklist  

### **6.1 No Partial Delivery**
Partial delivery is not acceptable.

### **6.2 No Substitutions**
Developer may not replace required features with alternatives.

### **6.3 No Deviation**
All implementation must match the Master Specification exactly.

# **SECTION 7 — MILESTONE‑BASED PAYMENT STRUCTURE**

Payments are tied to **verified delivery**, not effort or time.

Each Milestone must be:

- Fully delivered  
- Fully tested  
- Fully compliant  
- Fully validated  
- Fully confirmed in writing  

Only then is payment released.

# **SECTION 9 — PENALTIES FOR NON‑DELIVERY**

If Developer fails to deliver:

### **9.1 Payment Withholding**
Client may withhold payment for incomplete Milestones.

### **9.2 Contract Termination**
Client may terminate contract immediately.

### **9.3 Liability for Damages**
Developer may be held liable for:

- Lost time  
- Lost revenue  
- Replacement developer costs  
- Compliance risks  

### **9.4 No Refund of Completed Milestones**
Completed Milestones remain property of the Client.
