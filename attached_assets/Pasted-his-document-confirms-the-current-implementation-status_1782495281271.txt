2. GAPS THAT MUST BE FIXED BEFORE LAUNCH (MANDATORY)
Below are the 8 critical gaps that must be implemented to meet the full IBCCF specification.

GAP 1 — Missing “Case Created” Email to User
Required:  
When a case is submitted, the user must immediately receive:

“Your case has been registered”

Case ID

Expected next steps

Timeline for admin review

Status: Not implemented.

GAP 2 — Missing Admin Notification for New Case Submission
Required:  
Admins must receive:

Email alert

Dashboard notification

When:

A new case is submitted

A case is created manually

Status: Not implemented.

GAP 3 — Missing Admin Email When User Sends a Message
Required:  
When a user sends a message:

Admin must receive an email

Admin must receive a dashboard alert

Status: Only dashboard alert exists. Email missing.

GAP 4 — RBAC (Role-Based Access Control) Not Enforced
Required:  
Roles must be enforced:

super_admin → full access

admin → case/document management

agent → limited support actions

viewer → read-only

Status: All admins currently have full access.
This is a security risk.

GAP 5 — Missing Banned Keyword Moderation
Required:  
Community forum must include:

Configurable banned keyword list

Auto-flagging

Auto-logging

Admin moderation UI

Status: Not implemented.

GAP 6 — Missing Unified /health Endpoint
Required:  
Single endpoint returning:

DB status

Email SMTP status

OpenAI API status

Uptime

Version

Status: Only partial health checks exist.

GAP 7 — Missing Stage Transition Validation
Required:  
Admin must NOT be able to:

Skip stages

Move backwards

Jump arbitrarily

Server must enforce:

Only current_stage + 1 is allowed

Exceptions require super_admin override

Status: Not implemented.

GAP 8 — Tawk.to Integration Not Completed
Required:

Full integration

Fallback if script fails

Admin visibility of user messages

Countdown page embedding

Status: In progress, not complete.
