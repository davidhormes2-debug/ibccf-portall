1. INTRODUCTION
1.1 Purpose
This Software Requirements Specification (SRS) defines the complete functional, non‑functional, architectural, operational, and compliance requirements for the IBCCF platform.
It is intended for:

Developers

Admin/support teams

Auditors & compliance officers

Investors

Future hires

This document is legally binding as a representation of expected system behavior.

1.2 Scope
IBCCF is a regulated withdrawal, compliance, and case‑management platform consisting of:

Public case intake

Case holder portal

14‑stage withdrawal process

Document management

Compliance declarations

Countdown & reactivation enforcement

Admin dashboard

AI community forum

Automated email system

Security & audit framework

Monitoring & observability layer

This SRS defines every requirement necessary for full implementation.

1.3 Definitions
Case Holder — User with an active case.

Admin — Authorized staff managing cases.

Super Admin — Highest‑privilege role.

Countdown — Time‑bound compliance enforcement mechanism.

Reactivation — Payment‑based restoration of access after countdown expiry.

NDA Integrity Sweep — Automated daily verification of NDA compliance.

Declaration of Compliance — Legally binding user declaration.

3. FUNCTIONAL REQUIREMENTS
3.1 Public Case Intake
Users must be able to submit fraud complaints.

System must validate all fields.

System must generate Case ID.

System must send Case Created Email (mandatory).

System must notify admin (mandatory).

3.2 Access Code Workflow
Admin approves access code.

System emails access code.

Access code must be single‑use.

Access code must expire after 24 hours.

3.3 Case Holder Portal
3.3.1 Dashboard
Display stage progress.

Display next actions.

Display payout wallet.

Display timeline.

Display countdown (if active).

3.3.2 Document Uploads
Accept PDF/JPG/PNG only.

Max size 10MB.

Notify admin on upload.

Trigger automated email.

3.3.3 Declaration of Compliance
Multi‑section form.

Digital signature required.

Immutable after submission.

3.3.4 Messaging
User can send messages.

Admin receives email notification (mandatory).

User receives email when admin replies (mandatory).

3.3.5 Countdown
Display countdown timer.

Enforce lockout on expiry.

Reset withdrawal pathway.

Require reactivation deposit.

3.3.6 Reactivation
User uploads receipt.

Admin approves.

System restores access.

3.4 Admin Dashboard
3.4.1 Case Management
Search, filter, sort.

Edit case fields.

Change stages.

Impersonate user.

3.4.2 Document Review
Approve/reject documents.

Add rejection notes.

Trigger automated emails.

3.4.3 Email System
One‑off emails.

3.4.4 Security Controls

3.4.5 Countdown & Reactivation
