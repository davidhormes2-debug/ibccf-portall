### *Enterprise‑Grade Withdrawal, Compliance & Case‑Management System*

---

## **1. Overview**

IBCCF is a full‑stack, compliance‑driven case‑management and withdrawal‑processing platform designed for regulated environments.  
It includes:

- Public case intake  
- Case holder portal (14‑stage withdrawal process)  
- Document management  
- Compliance declarations  
- Countdown & reactivation enforcement  
- Admin dashboard  
- Automated email engine  
- AI‑powered community forum  
- Full audit logging  
- Security & monitoring layer  

This repository contains the **frontend**, **backend**, **infrastructure**, and **documentation** required to deploy and maintain the system.

# **3. Local Setup**

### **Step 1 — Clone Repository**
```
git clone https://github.com/ibccf/ibccf.git
```

### **Step 2 — Install Dependencies**
```
cd frontend && npm install
cd backend && npm install
```

### **Step 3 — Configure Environment Variables**
Copy `.env.example` → `.env` and fill in required values.

### **Step 4 — Run Database Migrations**
```
cd database
drizzle-kit push
```

### **Step 5 — Start Development Servers**
Frontend:
```
npm run dev
```
Backend:
```
npm run dev
```

# **9. Security Responsibilities**

Developers must:

- Enforce RBAC  
- Validate all inputs  
- Sanitize all user content  
- Protect against XSS  
- Protect against CSRF  
- Protect against SQL injection  
- Log all admin actions  
- Never store user data locally

# **10. Compliance Responsibilities**

Developers must ensure:

- Declaration immutability  
- NDA integrity  
- Withdrawal reset logic  
- Audit log completeness  
- GDPR‑aligned data handling

# **11. Incident Response**

If a developer identifies:

- Security vulnerability  
- Data integrity issue  
- Compliance breach  
- AI malfunction  
- Email failure  

They must:

1. Report immediately to project owner  
2. Document issue  
3. Apply hotfix  
4. Update audit logs
