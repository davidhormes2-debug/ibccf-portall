SECTION 7 — COUNTDOWN & REACTIVATION SYSTEM (FULL DETAIL)
The Countdown & Reactivation System is a regulatory‑grade enforcement mechanism designed to:

Ensure user compliance

Prevent abandonment of regulated withdrawal processes

Maintain audit integrity

Enforce session discipline

Reset non‑compliant cases

Require reactivation deposits

This system is mandatory and must operate with zero bypasses.

7.1 Countdown Activation (Admin Action)
Admin Inputs:
Target user

Duration (days, hours, minutes)

Reactivation deposit amount

Custom countdown message (optional)

System Actions:
Create countdown record in DB

Store start_time and end_time

Update user status → countdown_active

Invalidate all existing sessions

Force user to re-login

Display countdown banner on all portal pages

Trigger automated email:

“Countdown Activated — Portal Session Scheduled for Closure”

Log admin action in audit log

Security Requirements:
Only super_admin and admin roles may activate countdown

All countdown actions must be logged with:

Admin ID

Timestamp

IP address

Reason (if provided)

7.2 Countdown Display (User Portal)
UI Requirements:
Persistent countdown timer

Format: DD : HH : MM : SS

Visible on:

Dashboard

Upload pages

Declaration page

Messages

Case letter

Wallet view

Functional Requirements:
Timer must update every second

Timer must persist across:

Refresh

Logout/login

Device change

Timer must be server‑authoritative (no client manipulation)

Fallback Requirements:
If timer fails to load:

Display static warning

Prevent bypass

Log error to Sentry

7.3 Countdown Expiry (Zero Reached)
When countdown reaches zero, the system must execute the following atomically:

1. Force Logout
Terminate all active sessions

Rotate access code

Invalidate tokens

2. Reset Withdrawal Pathway
Set stage = 1

Clear stage‑specific validations

Reset all stage timestamps

3. Void Compliance Artifacts
Void NDA

Void Declaration of Compliance

Void any sealed settlement

Mark all prior validations as expired

4. Lock Portal Access
User status → reactivation_required

All portal routes redirect to Reactivation Page

5. Trigger Automated Emails
“Your Portal Has Been Reset Due to Inactivity”

“Reactivation Deposit Required”

6. Log Event
Countdown completion

Reset actions

IP address

Server timestamp
